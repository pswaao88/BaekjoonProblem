## 1. 문제 풀이 요약

### [흉부외과 또는 일반외과 의사 목록 출력하기] (Lv. 1)

- **핵심**: 특정 진료과에 속하는 의사 정보를 조회하고, 고용일자 및 이름순으로 정렬합니다.
- **주요 개념**: `SELECT`, `WHERE`, `OR`, `ORDER BY`, `DATE_FORMAT`
- **내 코드**:
  ```sql
  SELECT DR_NAME, DR_ID, MCDP_CD, DATE_FORMAT(HIRE_YMD, "%Y-%m-%d") 
  FROM DOCTOR
  WHERE MCDP_CD = 'CS' OR MCDP_CD = 'GS'
  ORDER BY HIRE_YMD DESC, DR_NAME ASC;
  ```

### [과일로 만든 아이스크림 고르기] (Lv. 1)

- **핵심**: 아이스크림 성분 정보를 활용해 특정 조건을 만족하는 아이스크림 맛을 조회합니다.
- **주요 개념**: `WITH` (CTE), `INNER JOIN`, `SELECT`, `WHERE`, `AND`
- **내 코드**:
  ```sql
  WITH ICECREAM AS (
      SELECT f.SHIPMENT_ID, f.FLAVOR, f.TOTAL_ORDER, i.INGREDIENT_TYPE 
      FROM FIRST_HALF AS f
      INNER JOIN ICECREAM_INFO AS i on f.FLAVOR = i.FLAVOR
  ) 

  SELECT FLAVOR FROM ICECREAM WHERE TOTAL_ORDER >= 3000 AND INGREDIENT_TYPE = 'fruit_based';
  ```

### [12세 이하인 여자 환자 목록 출력하기] (Lv. 1)

- **핵심**: 특정 나이 및 성별의 환자 정보를 조회하고, 전화번호가 없는 경우 'NONE'으로 표시합니다.
- **주요 개념**: `SELECT`, `WHERE`, `AND`, `IFNULL`, `ORDER BY`
- **내 코드**:
  ```sql
  -- 코드를 입력하세요
  SELECT PT_NAME, PT_NO, GEND_CD, AGE, IFNULL(TLNO, 'NONE') 
  FROM PATIENT WHERE AGE <= 12 AND GEND_CD = 'W' 
  ORDER BY AGE DESC, PT_NAME ASC;
  ```

### [조건에 맞는 도서 리스트 출력하기] (Lv. 1)

- **핵심**: 특정 연도에 출판된 특정 카테고리의 도서 정보를 조회합니다.
- **주요 개념**: `SELECT`, `WHERE`, `AND`, `YEAR()`, `DATE_FORMAT`, `ORDER BY`
- **내 코드**:
  ```sql
  -- SELECT * FROM BOOK;
  SELECT BOOK_ID, DATE_FORMAT(PUBLISHED_DATE, "%Y-%m-%d") FROM BOOK WHERE YEAR(PUBLISHED_DATE) = 2021 and CATEGORY = '인문' ORDER BY PUBLISHED_DATE;
  ```

### [인기있는 아이스크림] (Lv. 1)

- **핵심**: 총주문량을 기준으로 아이스크림 맛을 내림차순으로 정렬하여 조회합니다.
- **주요 개념**: `SELECT`, `ORDER BY`
- **내 코드**:
  ```sql
  SELECT FLAVOR FROM FIRST_HALF ORDER BY TOTAL_ORDER DESC, SHIPMENT_ID ASC;
  ```

## 2. 주요 개념 및 심화 학습 포인트

오늘 풀이에서 중요했거나, 비슷한 함수 설명, 추가적으로 학습이 필요한 개념, 같은 문제 다른 풀이 들을 정리합니다.

### [핵심 개념 1: JOIN]

- **개념**: 두 개 이상의 테이블을 공통된 열(column)을 기준으로 연결하여 데이터를 검색하는 방법입니다. 관계형 데이터베이스의 핵심 기능 중 하나입니다.
- **언제 사용하는가?**: 정규화를 통해 분리된 테이블들의 데이터를 조합하여 의미 있는 결과셋을 만들어야 할 때 사용합니다. 예를 들어, 아이스크림의 기본 정보와 성분 정보가 다른 테이블에 나뉘어 있을 때, 이를 합쳐 '과일로 만든 아이스크림'을 찾는 경우입니다.
- **사용 예시**: 
  ```sql
  -- FIRST_HALF 테이블과 ICECREAM_INFO 테이블을 FLAVOR 기준으로 합침
  SELECT
      F.FLAVOR,
      F.TOTAL_ORDER,
      I.INGREDIENT_TYPE
  FROM FIRST_HALF AS F
  INNER JOIN ICECREAM_INFO AS I
      ON F.FLAVOR = I.FLAVOR;
  ```
- **학습 포인트**: `INNER JOIN`은 두 테이블에 모두 존재하는 데이터만 가져옵니다. `LEFT JOIN`, `RIGHT JOIN`, `FULL OUTER JOIN` 등 다양한 JOIN 방식의 차이점과 사용 사례를 학습하면 좋습니다.
- **관련 문제**: [과일로 만든 아이스크림 고르기]

### [핵심 개념 2: 날짜 함수 (DATE_FORMAT, YEAR)]

- **개념**: 날짜 및 시간 데이터를 원하는 형식으로 바꾸거나, 특정 부분(년, 월, 일 등)을 추출하는 함수입니다.
- **언제 사용하는가?**: 날짜를 특정 형식(예: 'YYYY-MM-DD')으로 출력해야 하거나, 특정 연도나 월에 해당하는 데이터만 필터링하고 싶을 때 사용합니다.
- **사용 예시**:
  ```sql
  -- HIRE_YMD를 '년-월-일' 형식으로 포맷팅
  SELECT DATE_FORMAT(HIRE_YMD, '%Y-%m-%d') FROM DOCTOR;

  -- PUBLISHED_DATE에서 연도만 추출하여 2021년인지 확인
  SELECT * FROM BOOK WHERE YEAR(PUBLISHED_DATE) = 2021;
  ```
- **학습 포인트**: `DATE_FORMAT`의 다양한 포맷 코드( `%Y`, `%y`, `%m`, `%d`, `%H`, `%i`, `%s` 등)를 익혀두면 원하는 형태로 날짜를 자유롭게 표현할 수 있습니다. `MONTH()`, `DAY()`, `DATE()` 등 다른 날짜/시간 함수들도 함께 학습하면 유용합니다.
- **관련 문제**: [흉부외과 또는 일반외과 의사 목록 출력하기], [조건에 맞는 도서 리스트 출력하기]
